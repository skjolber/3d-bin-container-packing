### Key asymmetry
- 3D moveToZZ already uses insertionSortThis, but moveToXX and moveToYY still use sortThis
- 2D moveToXX/moveToYY use plain IntArrayList (not CustomIntArrayList), so no insertionSortThis
- 2D values.sort() always uses Arrays.sort regardless of list size

### Conclusion
Three independent optimizations:
A. 3D: insertionSortThis for moveToXX and moveToYY (2-line change)
B. 2D: Change moveToXX/moveToYY to CustomIntArrayList + insertionSortThis
C. 2D: Add insertion sort threshold in Point2DFlagList.sort(Comparator, int) for endIndex ≤ 32
