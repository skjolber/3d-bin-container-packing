=== CHOSEN OPTIMIZATION ===

Target: DefaultPointCalculator2D.removeEclipsed() inner loop

The inner loop (index = limit..size-1) iterates over EXISTING points.
Code analysis proves these items are ALWAYS unflagged when removeEclipsed() is called:
  1. Line 437: values.removeFlagged() is called -- all remaining items are unflagged
  2. Line 449: values.move(added) shifts items right, preserving flags (all false)
  3. Lines 450-451: setAll() explicitly sets flags=false for new items
  4. Line 453: removeEclipsed(added) runs

Therefore, the check if(values.isFlag(index)) continue; in the inner loop (index >= limit)
is ALWAYS false and safely removable.

Also hoisting unsortedMinX and unsortedArea as local variables eliminates repeated
pointer chasing in the tight inner loop.
